The backbone of WenZi is a massive, meticulously curated dataset, aggregated from premier linguistic sources and refined through a multi-stage pipeline.

[sidebar]

* icon:database[] *CC-CEDICT* - Core Chinese-English dictionary.

[.text-center]
== 🏗 System Architecture

WenZi is built on a modern, scalable, and resilient technology stack, designed for high performance and a seamless user experience.

.System Architecture Diagram
[plantuml, system-architecture-diagram, png]
----
@startuml
!theme vibrant
skinparam rectangle<<backend>> { backgroundColor #444; fontColor #FFF; }
skinparam database<<db>> { backgroundColor #888; fontColor #FFF; }

cloud "User's Device" as UserDevice
package "Cloud Platform" {
    node "Frontend (Astro + React)" as Frontend
    node "API Gateway" as Gateway
}
package "Backend Services" <<backend>> {
    node "Search Engine" as Search
    node "Dictionary Engine" as Dictionary
    node "OCR Engine" as OCR
}
package "Data Tier" {
    database "Primary DB (Turso)" as SQLDb <<db>>
    database "Vector DB (Pinecone)" as VectorDb <<db>>
    database "Static JSON Chunks" as StaticJson <<db>>
    database "Cache (Redis)" as Cache <<db>>
}

UserDevice --> Frontend
Frontend --> Gateway
Gateway --> Search & Dictionary & OCR
Search --> StaticJson & VectorDb & Cache
Dictionary --> SQLDb & StaticJson
OCR --> VectorDb
@enduml
----

[%collapsible]
.Dive Deeper into the Architecture
====
The system follows a microservices-inspired approach with a polyglot persistence strategy.

*   **Frontend:** Built with Astro for fast, static-first pages, with interactive React "islands" for dynamic components. Styled with Tailwind CSS.
*   **Backend:** A set of serverless functions acting as specialized engines (Search, OCR, etc.).
*   **Databases:** We use Turso (libSQL) for relational user data, Pinecone for vector search (OCR/semantic), Redis for caching, and sharded JSON files for the main dictionary data to ensure lightning-fast lookups.
====

'''

[.text-center]
== ⚙️ Core Engines Deep Dive

[cols="2,3"]
|===
| [^.^a]
.🔍 *Search Engine*
--
Our custom search service is optimized for the complexities of the Chinese language.

[checklist]
* Parses Pinyin, Hanzi, English.
* Fuzzy matching for typos.
* Radical/component-based search.
--

| [^.^a]
[source,typescript,linenums]
----
// src/lib/searchService.ts
export class SearchService {
  async search(query: string, opts: SearchOptions) {
    const parsed = this.parseQuery(query); // <1>
    let candidates = await this.fetchFromChunks(parsed); // <2>
    const ranked = this.rank(candidates, parsed); // <3>
    return ranked.slice(0, opts.limit);
  }
}
----
<1> Normalize Pinyin, tones, etc.
<2> Fetch candidates from pre-sharded JSON for speed.
<3> Rank results by relevance.

| [^.^a]
.🈶 *Hanzi Engine*
--
Provides rich, structured data for any character.

[dlist]
`simplified`:: e.g., 你
`traditional`:: e.g., 你
`pinyin`:: nǐ
`hsk_level`:: 1
`stroke_count`:: 7
--

| [^.^a]
[TIP]
.Character Breakdown
====
The Hanzi engine decomposes characters into their fundamental components (radicals), helping learners understand character construction and etymology. For example, the character `好` (hǎo) is broken down into `女` (nǚ - woman) and `子` (zǐ - child).
====
|===
